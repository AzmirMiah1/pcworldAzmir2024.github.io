Storing Program Concept and Computer Architectures

The history of computing changed with the notion of storing programs in the memory so that the computers can utilized in sequence to execute instructions and store it for the future use. This innovation, credited to Von Neumann Architecture, enables a computer to run program and data in the same memory which of course makes the system more flexible and easier to program.


Neumann Architecture

  • Definition: The Von Neumann architecture uses a single memory space to store both data and instructions.

  • Features:
    • Shared Memory: Both data and instructions are stored in the same RAM.
    • Single Data Bus: Data and instructions share the same communication path.
    • Sequential Processing: The system processes one instruction at a time (fetch-decode-execute cycle).

Harvard Architecture

  • Definition: The Harvard architecture separates memory for data and instructions.

  • Features:
    • Separate Memories: Different storage for instructions and data.
    • Dual Data Buses: Instructions and data have separate communication paths.
    • Parallel Processing: Enables simultaneous fetching of instructions and data.

Advantages and Disadvantages

The concept of storing program which is seen in the Von Neumann Architecture pattern has a lot of benefits. Since the data and instructions are stored in the same location, hardware implementation is made easier and since the instructions can be easily reprogrammed it is easier to employ the computer for other tasks. This design is affordable and also becomes the foundation for most current computing structures.

However, it has also major drawbacks like the Von Neumann bottleneck. It is so because data and instructions use the same bus for passing information and thus they cannot access the bus at the same instance thus slowing down the speed of the system. Furthermore, it makes it easier for errors such as instructions being written over by data already residing in the shared memory.

Operating Systems: Purposes, Features, and Functions

An operating system is a vital piece of system software that manages hardware resources, provides a user interface, and enables applications to run.


  • Purposes of an OS:
    • Acts as a bridge between the user and hardware.
    • Ensures efficient use of system resources.
    • Provides a platform for running application software.

  • Features of an OS:
    • User Interface: Provides the end user with a GUI (Graphical User Interface) or a CLI (Command Line Interface).
    • Multitasking: Enables several applications at a time to execute.
    • Memory Management: Manages the physical RAM in the system either dividing it into portions or managing the overall amount assigned.
    • Device Management: Responsible for managing devices such as printers, keyboards, and the USB flash drives.
    • Security: Provides a defense mechanism in preventing unauthorized access to its resources.

  • Functions of an OS:
    • Process Management: Monitors and time manages activities to guarantee optimality in flow of run.
    • File System Management: Responsible for storage of files on storage devices and also retrieval of files kept on storage devices.
    • Network Management: Responsible for managing connections through the networks.
    • Error Handling: Identifies systems failures and errors and works to rectify them.

Range of Common Utiliy Software

  • Antivirus Software: Protects against malware and viruses (e.g., Norton, McAfee).
  • Disk Cleanup Tools:Removes unnecessary files to free up storage.
  • Backup Software: Creates copies of data and information. It is very helpful in the event of dataloss.
  • Compression Tools: Reduces file sizes for easier storage and transfer (e.g., WinRAR, 7-Zip).
  • Firewall Utilities: Monitors and controls incoming/outgoing network traffic.
  • Defragmentation Tools: Reorganizes fragmented files on a disk for faster access.

Types of Application Software

Application software can be best defined as those programs that are used to perform tasks by the users; some of examples are word processing software, web browsing software, photo editing software, etc. They are designed to run over the operating system.


  • Productivity Software: Used to enhance workplace efficiency, including word processors (e.g., Microsoft Word), spreadsheet programs (e.g., Excel), and presentation software (e.g., PowerPoint).
  • Media Editing Software: Programs for creating and editing multimedia content, such as Adobe Photoshop for image editing, Adobe Premiere for video editing, and Audacity for audio production.
  • Web Browsers: Software to access the internet, enabling users to browse websites and access online services. Examples include Google Chrome, Mozilla Firefox, and Safari.
  • Communication Software: Enables communication and collaboration, such as email clients (e.g., Microsoft Outlook) and messaging platforms (e.g., Slack and Zoom).
  • Games: Applications for entertainment, available on various platforms like PCs, consoles, and mobile devices.

Purpose of BIOS

The BIOS is the firmware located in the motherboard of a computer. It is also used during booting to initialize the hardware and also offer run time support to the operating system. When the computer is switched on, BIOS runs a set check referred to as the Power-On Self-Test checking the RAM, CPU and other components.

Furthermore, it brings into functionality the devices like keyboards, monitors, and storage devices among others. Through BIOS setup utility users are able to control different system setting including boot order. The primary functions of BIOS are to serve as the connection between the hardware and the operating system, the computer booting and to manage an efficient running of the computer.
